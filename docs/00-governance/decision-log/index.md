# Decision Log Index

This index lists every numbered creative decision for **The Unnecessary**, with its status, category, a short summary, and a link to the full entry. The Decision Log records what was decided, what alternatives were rejected, and why, so that another writer or AI does not unknowingly rebuild an earlier version of the story. It preserves the reasoning behind canon; it does not establish canon. The Story Bible, Character Bible, Master Timeline, Technology Rules, and Plot Outline establish current canon.

Read a single decision file when a proposed idea resembles something previously decided, when someone wants to revise major canon, or when two active documents appear to conflict. Each decision lives in its own file under `decisions/`.

## How to use this index

- To check whether an idea was already settled or rejected, scan the table below and open the matching decision file.
- To confirm what must not be casually reintroduced, see [rejected-concepts.md](./rejected-concepts.md).
- To see what is still unresolved and must not be treated as fixed canon, see [open-decisions.md](./open-decisions.md).
- For broader process and conflict-handling authority, see the [novel development guide](../novel-development-guide.md).

## Decision Statuses

The following legend defines the status labels used in the table and in each decision file.

### Locked for Current Draft

The decision should be treated as canon while developing the current version of the novel.

It can still be changed deliberately, but should not be casually contradicted.

### Locked for Current Workflow

The decision governs the current development workflow, tooling, and document organization rather than story canon. It should be treated as fixed while that workflow is in use, and revised deliberately only when the workflow itself changes.

### Active but Revisable

The decision currently guides development, but may change without requiring a complete reconstruction of the project.

### Provisional

The decision is useful for current planning but has not been fully tested.

### Rejected

The idea was deliberately removed and should not be reintroduced without a strong reason.

### Superseded

The decision was replaced by a later decision.

## Decision entry template

New decisions follow a fixed entry format covering Decision, Previous or Alternative Direction, Reason, Consequences, Affected Documents, and Reconsider Only If. The authoritative template is preserved in the [novel development guide](../novel-development-guide.md) and in the source decision log; match it exactly when adding an entry.
