# Style Guide Index

This directory holds the Prose and Style Guide for The Unnecessary, split into eleven independently loadable files so a chapter task can load only the style rules it needs. The guide defines how the novel should sound and feel on the page. It governs voice, viewpoint, distance, dialogue, exposition, pacing, formatting, and the habits to avoid.

Authority and status: every file here is active-canon under the style-rule authority. The Style Guide version block (Version 1.0, last updated 2026-06-24) is preserved at the top of core-prose.md. For canonical facts about characters, technology, and the world, defer to the canon authorities under `../../20-canon/`.

## Read first

Read `core-prose.md` first. It carries the original Style Guide version block, the purpose statement, the core prose identity, the central stylistic principle, and the final style standard. It frames everything in the other ten files.

## Files

| File | Summary | Load when |
| ---- | ------- | --------- |
| [core-prose.md](core-prose.md) | Core prose identity, central stylistic principle, sentence style, punctuation, diction, description, environmental tone, free indirect style, and internal thought. Holds the Style Guide version block and final style standard. | Starting any chapter, or setting overall sentence rhythm, punctuation, diction, and description. |
| [viewpoint.md](viewpoint.md) | Narrative perspective, person and tense, viewpoint characters, one viewpoint per chapter, no omniscient leakage, and narrative distance. | Choosing a chapter viewpoint, checking distance, or fixing omniscient leakage. |

| [technology-in-prose.md](technology-in-prose.md) | Explain through need, the three-layer technical rule, avoiding lecture dialogue and magical interfaces, and worldbuilding exposition. | Rendering a technical problem, a repair, or worldbuilding exposition. |

| [formatting.md](formatting.md) | Chapter headings, dates and locations, numbers, time, italics, interface text, names and terminology, and profanity. Also the project's single house **style sheet**: it carries the copy-editor's Copy-Edit Consistency Ledger (house spellings, hyphenation, recurring-term treatment with a pronunciation column for the audio agents, a homophone watch-list, and a decisions log). | Formatting a chapter, an interface message, checking a name or number, or pinning a house spelling or recurring-term treatment. |
| [prohibited-patterns.md](prohibited-patterns.md) | Prose habits to avoid plus science-fiction, dystopian, and character clichés. | Self-editing for generic AI patterns or genre clichés. |

## Common tasks

- Draft a new chapter: read core-prose.md, viewpoint.md, the relevant character voice in character-voices.md, and pacing-and-structure.md.
- Write a technical or repair scene: read technology-in-prose.md and ai-dialogue.md.
- Write an emotional or moral confrontation: read emotion-and-moral-content.md and dialogue.md.
- Self-edit a draft: read prohibited-patterns.md and revision-checklist.md.
- Format or name-check a chapter: read formatting.md.
- Brief another AI to draft: read revision-checklist.md.
