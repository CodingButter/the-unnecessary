# Premise Framing Routing Note

The Phase 08 coverage sweep flagged four Story Bible headings as possibly unrouted:
"The Core Mystery", "The Moral Problem", "Original Purpose", and "Main Locations".
On review, none of the four is a genuine premise-framing gap. Each already has an
active canon destination, so this file carries no duplicated canon and only records
where each section is owned.

## Classification Summary

| Story Bible heading | Category | Owning destination |
| --- | --- | --- |
| Main Locations | (a) genuine standalone world section, but it has an active home in the locations domain, not premise framing | `./locations/index.md` |
| Original Purpose | (b) summarize-and-linked subsection of the Morrow technology overview | `../technology/ai/morrow.md` |
| The Moral Problem | (b) summarize-and-linked subsection of the Morrow technology overview | `../technology/ai/morrow.md` |
| The Core Mystery | (b) summarize-and-linked subsection of the Morrow technology overview | `../technology/ai/morrow.md` |

## Where Each Section Is Owned

### Main Locations

"Main Locations" is a top-level world and setting section, a sibling of the premise,
social-structure, and Mars sections. It is not a subsection of a character or
technology overview, so it qualifies as category (a). It is not, however, premise
framing. Its content belongs to the locations domain, and every one of its
subsections already has a dedicated file. Routing it here would let this file
silently do a locations file's job, which is not permitted.

- Locations index: `./locations/index.md`
- Eli's Neighborhood: `./locations/elis-neighborhood.md`
- Lakeward: `./locations/lakeward.md`
- Northglass: `./locations/northglass.md`
- The Neighborhood Network: `../technology/infrastructure/community-infrastructure.md`
- Mars: `./locations/mars-sites.md`
- Greater Detroit primary setting: `./locations/greater-detroit.md`

### Original Purpose, The Moral Problem, The Core Mystery

These three are `###` subsections nested under the Story Bible's `## Morrow`
technology overview, alongside Architecture, Foundational Principles, Personality,
and Why Asterion Wants It. They are intentionally summarize-and-linked overview
subsections under the Phase 12 model and are therefore not gaps. Their canon is
owned by the Morrow technology and character files.

- Morrow technology canon: `../technology/ai/morrow.md`
- Morrow character profile: `../characters/profiles/morrow.md`
