# Canon Index

This is the entry point to the established canon of the novel "The Unnecessary". Canon is split into four domains, each with its own index. Load the domain index you need rather than the whole canon. Each domain index summarizes its files and links to the authority; the authority file always wins over any summary.

Read first: the domain index for the subject you are working on.

| Domain | Authority over | Read when | Index |
| --- | --- | --- | --- |
| World | Premise, society, infrastructure decline, enclaves, Mars overview, locations, broad arc, series direction, consistency rules | Setting, world facts, locations, the shape of society, the broad Book One arc | [world/index.md](world/index.md) |
| Characters | The thirteen character profiles plus the relationship map and viewpoint and dialogue rules | Any character question: history, motive, voice, secret, arc, relationships | [characters/index.md](characters/index.md) |
| Technology | AI foundations, Crown, Morrow, infrastructure systems, robotics, medicine, transport, security, Mars technology, Northglass facility, hard plot restrictions, failure rules | Any capability or limit question; what a system can and cannot do | [technology/index.md](technology/index.md) |
| Timeline | Dates and chronology: historical eras, character birth dates, the Book One day-by-day timeline, knowledge and secret timelines | Any "when" question; ordering of events; what a character knows by a given date | [timeline/index.md](timeline/index.md) |

## Authority and status

All four domains hold active canon, authoritative by subject. Approved manuscript prose, when it exists, outranks these files for events it depicts; see [../00-governance/canon-hierarchy.md](../00-governance/canon-hierarchy.md) for the full ranked order and the per-domain ownership rules.

## Cross-domain rule

A concept that crosses domains has one owner per fact type and is linked, never duplicated. For example, Morrow's behavioral identity lives in [characters/profiles/morrow.md](characters/profiles/morrow.md), its architecture and capabilities in [technology/ai/morrow.md](technology/ai/morrow.md), and its Book One progression in the plot files. Each canon file summarizes a cross-domain concept in a sentence and links to the owner.
