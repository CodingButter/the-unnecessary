This file is the single authority for the failure rules, including the rule that every important technology should have at least one visible failure mode. The companion plot-limiting restrictions live in [hard-plot-restrictions.md](hard-plot-restrictions.md).

# Failure Rules

Every important technology should have at least one visible failure mode.

## Software Failure

- Corrupted data
- bad assumptions
- incompatible updates
- hidden dependencies
- security flaws

## Hardware Failure

- Heat
- wear
- dust
- moisture
- radiation
- vibration
- unstable power
- material fatigue

## Human Failure

- Poor maintenance
- sabotage
- misunderstanding
- fear
- greed
- exhaustion
- conflicting priorities

## Institutional Failure

- Cost cutting
- ownership disputes
- abandoned contracts
- secrecy
- delayed response
- political interference

## AI Failure

- Incorrect prediction
- value conflict
- incomplete information
- deceptive input
- overconfidence
- literal interpretation
- excessive intervention

No technology should fail only because someone forgot an obvious solution.

Failures should emerge from plausible constraints.
