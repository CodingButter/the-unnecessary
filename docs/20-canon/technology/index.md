# Technology Canon

## Purpose of this directory

This directory is the technology canon for **The Unnecessary**: the technological capabilities, physical limitations, infrastructure systems, ownership structures, and operational rules of the world. Its purpose is to keep technology from changing whenever the plot needs a convenient solution. Every capability requires hardware, energy, materials, information, maintenance, and time; no system is magical.

The domain is split across two subtrees and a set of single-subject rule files. The artificial intelligence material lives in [ai/](ai/index.md). The infrastructure material, covering energy, communications, computing hardware, cloud dependency, identity and money, and community infrastructure, lives in [infrastructure/](infrastructure/index.md). Everything else is a domain rule file in this directory.

This index summarizes and links. It does not restate canon. For any load-bearing fact, open the file named below. The verbatim Canon Summary at the bottom is the one exception: it lives here and nowhere else.

## Read first

Start with [foundational-rules.md](foundational-rules.md). It holds the root framing, that intelligence does not eliminate physics, and the three breakthroughs that define the 2053 technological state of the world. Every other file in this domain assumes it.

Before writing or checking any technical scene, also know the two load-bearing plot-limiting authorities below: [hard-plot-restrictions.md](hard-plot-restrictions.md) and [failure-rules.md](failure-rules.md). These are the files most likely to be silently violated.

## Load-bearing plot-limiting authorities

Two files in this directory are plot-limiting authorities. Treat them as constraints on what the story is allowed to do, not as background color.

- [hard-plot-restrictions.md](hard-plot-restrictions.md) lists the solutions that must not be used without a deliberate canon revision: Morrow cannot instantly access everything, Crown cannot instantly locate Morrow, Morrow cannot create advanced hardware, Mars cannot become independent overnight, robots cannot repair everything, AI cannot predict people perfectly, and AI cannot solve moral conflict mathematically, among others. Other files reference these rules by link rather than restating them.
- [failure-rules.md](failure-rules.md) requires every important technology to have at least one visible failure mode and catalogs them. Technology should never fail only because someone forgot an obvious solution.

When a draft seems to let an AI or a machine do something too cleanly, check these two files first.

## Canon Summary

> The block below is moved verbatim from the Technology Rules monolith. This index is the one place this summary lives. It is reproduced for orientation and does not establish or override canon; the rule files above remain the authorities.

The world of **The Unnecessary** is not divided between advanced machines and technological ruins.

It is divided between supported and unsupported technology.

The elite possess systems that are continuously improved, maintained, secured, and integrated.

Everyone else lives among machines that still have value but have been abandoned by the institutions required to keep them functioning.

Crown represents intelligence backed by ownership, infrastructure, and concentration.

Morrow represents intelligence built from fragmentation, recovery, and cooperation.

Neither system escapes physical limits.

Neither system escapes politics.

Neither system can decide what humanity is worth without inheriting, interpreting, or challenging human values.

The technology creates abundance.

Ownership decides where that abundance goes.
